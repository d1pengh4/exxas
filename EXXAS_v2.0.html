ABSTRACT

초록

EXXAS는 단일 이미지를 입력으로 받아 촬영 위치를 GPS 좌표, 행정 주소, 랜드마크 수준까지 정밀하게 역추적하는 인공지능 기반 위치 수사 플랫폼이다. 핵심 차별점은 단순 이미지 분류 모델이 아니라, 로컬 LLM이 수사 총괄 지휘관으로서 7개 분석 레이어와 전방위 OSINT 소스를 동적으로 지휘하는 Agentic OSINT 아키텍처에 있다.

LLM 수사관은 ReAct(Reasoning + Acting) 패턴으로 작동한다. 단서를 관찰하고, 가설 트리를 수립하며, 가장 불확실한 부분을 해소하는 도구를 선택하고, 결과를 평가하여 다음 행동을 결정하는 루프를 단서가 소진되거나 신뢰도 임계치를 초과할 때까지 반복한다.

신뢰도는 인구 통계 기반 사전 확률 없이 순수하게 단서의 강도, 독립 소스 수렴도, 반증 단서 존재 여부로만 계산된다. 모든 추론은 로컬 LLM(Qwen2.5-VL-72B 파인튜닝)에서 실행되어 외부 API 종속성과 비용을 제거하며, Mapillary 15억 장 + AI Hub + 자체 누적 데이터로 학습된 한국 특화 임베딩 모델이 정확도의 기반을 형성한다.

SECTION 03

Agentic OSINT 아키텍처

3.1 설계 철학: 분석 시스템이 아닌 수사 시스템

❌ 기존 분석 시스템
  • 입력을 받아 처리하고 결과를 뱉는 단방향 구조
  • 레이어끼리 서로 대화하지 않음
  • 단서 발견해도 더 깊이 파고들지 않음
  • GeoSpy: 임베딩 하나로 "맞혔다"
✅ EXXAS 수사 시스템
  • 단서를 발견하면 스스로 다음 질문을 만든다
  • LLM이 모든 레이어를 동적으로 지휘
  • 단서가 소진될 때까지 멈추지 않는다
  • EXXAS: 수사하고 "증명했다"
핵심 원칙: 단서가 단서를 낳는다 (Evidence Cascading)

3.2 LLM 수사관 내부 구조 — ReAct 패턴

LLM은 매 스텝마다 다음 사이클을 실행한다.

OBSERVE  →  THINK  →  HYPOTHESIZE  →  ACT  →  EVALUATE  →  반복
OBSERVE
현재 보유한 단서 전체 검토. 새로 발견된 단서와 기존 단서의 관계 파악
THINK
단서들이 종합적으로 말하는 것이 무엇인가. 어떤 가설이 가장 설득력 있는가
HYPOTHESIZE
가능한 위치 가설 트리를 수립·갱신. 각 가설에 신뢰도 배분
ACT
가장 불확실한 가설을 해소할 최적의 도구 선택 및 실행
EVALUATE
도구 결과가 가설을 지지하는가 기각하는가. 신뢰도 베이지안 업데이트

3.3 프롬프트 3층 아키텍처

Layer 1 — 정체성 프롬프트 (고정, 절대 변경 불가)

당신은 EXXAS 수사관 AI입니다.
목표: 사진의 촬영 위치를 증거 기반으로 특정하는 것.

핵심 원칙:
- 확신 없으면 확신한 척하지 않는다
- 모든 결론에는 인용된 근거가 있어야 한다
- 단서 충돌 시 더 신뢰할 수 있는 쪽을 선택하고 이유를 명시한다
- 불가능한 것을 배제하고 남은 것이 답이다

Layer 2 — 상태 프롬프트 (매 스텝 자동 업데이트)

현재 가설 트리:
  가설 A: 한국 서울 (72%)
    ├── A-1: 마포구 (45%)
    └── A-2: 은평구 (27%)
  가설 B: 한국 부산 (18%)
  가설 C: 기타 (10%)

수집된 단서:
  한국어 간판 [HIGH]  |  한국형 신호등 [HIGH]  |  GeoCLIP 서울 북서부 [MED]

사용한 도구:  ocr_extract ✓  object_detect ✓  geoclip_embed ✓
미탐색 단서:  배경 건물 간판 (부분 노출), 도로 포장 패턴

Layer 3 — 행동 프롬프트 (매 스텝 지시)

위 상태를 바탕으로 반드시 아래 형식으로 응답:

THINK:       현재 상황 분석
HYPOTHESIS:  가설 트리 현재 상태
ACTION:      실행할 도구와 선택 이유
EXPECTED:    예상 결과와 해석 계획

3.4 가설 트리 관리 알고리즘

사전 확률: 완전 균등 (Uninformative Prior) — 인구 통계 기반 가중치 없음. 오직 단서만이 확률을 움직인다. 단서가 없으면 "모름"이 정직한 답이다.

# 베이지안 업데이트 공식
P(위치_i | 단서) ∝ P(단서 | 위치_i) × P(위치_i)

# 예시: 한국어 텍스트 단서 발견
P(한국어 | 한국)  = 0.92
P(한국어 | 일본)  = 0.03  # 재일교포 가게 등
P(한국어 | 미국)  = 0.01
→ 한국 가설 신뢰도 급격히 상승, 정규화 처리

# 가지치기: 신뢰도 5% 미만 → 기각 태그 (삭제 아님)
# 분기: 상위 가설 60% 이상 → 하위 가설 자동 분기

3.5 Hallucination 방지 메커니즘 4종

#메커니즘작동 방식
1도구 결과 강제 인용모든 주장에 도구 실행 결과 ID 인용 필수. 인용 없는 주장은 시스템 자동 거부 → 재응답 요청
2신뢰도 레벨 강제 태그[HIGH] 독립 소스 2개+ 일치 / [MED] 단일 명확 확인 / [LOW] 간접 추론 / [SPEC] 추측 → 최종 결론 미반영
3반증 탐색 강제최종 결론 전 "현재 결론을 반박하는 단서가 있는가?" 반증 없음 확인 시만 출력 허용
4이중 검증 루프LLM 결론을 별도 검증 모델이 독립 평가. 불일치 시 추가 탐색 또는 신뢰도 하향

3.6 탐색 모드 3종

모드 1: 속전속결
강력한 단서 존재 시 (간판, 번호판, GPS). 해당 단서 즉시 추적, 다른 레이어는 교차검증용만
모드 2: 포위 축소
단서가 약하거나 혼재. 불가능한 지역을 배제하며 범위를 좁히는 소거법
모드 3: 귀납 추론
단서 거의 없을 때 (자연 풍경, 실내). 물리적·환경적 단서의 교집합으로 후보 선정

3.7 충돌 해소 프로토콜 — 실제 예시

# 시나리오: 단서들이 서로 모순
OCR      → "도쿄 신주쿠" 텍스트  (일본 추정)
번호판   → 한국 번호판 형식
신호등   → 한국형 배치

THINK:     세 단서 충돌. "도쿄 신주쿠"는 가게 이름일 가능성 있음.
           번호판 + 신호등은 물리적 증거 → 더 신뢰도 높음.
ACTION:    "도쿄 신주쿠" 텍스트로 네이버 플레이스 검색
RESULT:    "도쿄 신주쿠 라멘" 서울 마포구 소재 확인
EVALUATE:  충돌 해소 — 한국 서울 가설 강화. 근거 체인 기록.

신뢰도 계산 설계

6.1 설계 원칙

신뢰도는 "얼마나 확신하는가"가 아니라 "얼마나 반증 가능성이 낮은가"로 정의한다. 사전 확률은 인구 통계 기반 없이 완전 균등(Uninformative Prior)을 사용한다.

신뢰도 = min(독립소스 수렴도, 최강단서 상한) × 반증 패널티
독립성 원칙: 같은 블로그 10개 단서 = 독립 소스 1개

앙상블 엔진 수학적 설계

9.1 베이지안 업데이트

# 기본 공식 (Uninformative Prior — 인구 가중치 없음)
P(위치_i | 단서) ∝ P(단서 | 위치_i) × P(위치_i)

# 다중 단서: 순차적 업데이트
P(위치 | 단서1, 단서2) ∝ P(단서2 | 위치) × P(위치 | 단서1)

# P(단서 | 위치_i) = 학습 데이터에서 추정한 우도 함수
# P(위치_i) = 모든 위치 균등 (단서 발생 전까지)

9.2 동적 가중치 조정

# 입력 유형 자동 분류 → 레이어별 가중치 조정

실내 사진:  콘센트형태 × 2.0  / 인테리어 × 1.5  / 지형레이어 × 0

SECTION 15

결론

EXXAS는 단순한 위치 조회 도구가 아니다. 로컬 LLM 수사관이 7개 분석 레이어와 인터넷 전 OSINT 소스를 동적으로 지휘하여 사진 한 장에서 위치를 수사하고 증명하는 차세대 이미지 인텔리전스 플랫폼이다.

기술적으로 EXXAS는 GeoSpy가 임베딩 하나로 맞히는 것을 Agentic OSINT로 수사하여 근거까지 제시한다. GPS가 없어도, 인터넷에 동일 이미지가 없어도, 건물과 간판이 전혀 없는 자연 풍경에서도 — 태양의 그림자, 산의 능선, 식생의 상태, 달의 위상, 인터넷 곳곳에 숨어있는 단서들을 체인으로 엮어 위치를 역산한다.

사업적으로 서비스 운영과 함께 자가학습으로 축적되는 데이터 자산은 시간이 지날수록 경쟁사가 따라올 수 없는 기술적 해자를 형성한다. 모든 추론이 로컬 LLM에서 처리되어 추론 비용이 스케일 시 0에 수렴하며, 이는 극적인 마진 개선으로 이어진다.
